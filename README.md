# My-SQL-Project-Repository
## Project Summary
This repository contains my SQL project completed during the Data Technician Bootcamp (May–June 2026).
Across multiple real‑world tasks, I used SQL to query, clean, and analyse data from retail, sales, and global datasets using MySQL Workbench.

The project demonstrates essential SQL skills used by data technicians and analysts to extract insights, support business decisions, and work with relational databases.

## Project Overview
Throughout the SQL module, I worked with several datasets, including:

Northwind Traders (retail & sales data)

World Database (countries, cities, languages)

SkillUp Academy scenario (database design & normalisation)

The tasks involved writing SQL queries to retrieve, filter, sort, group, and join data across multiple tables.

## Key SQL Skills Demonstrated
### Data Retrieval with SELECT
I wrote queries to extract specific columns, full tables, and filtered subsets of data, such as:

Customer names and cities

Product lists

City and country information

High‑value products and recent orders

These queries formed the foundation for deeper analysis.

### Filtering Data with WHERE
I used the WHERE clause to filter records based on:

Price ranges

Country or city

Year of order

Population thresholds

Text patterns using LIKE

This allowed me to answer targeted business questions, such as identifying high‑value customers or cities with specific characteristics.

### Sorting & Ranking with ORDER BY
I applied ORDER BY to sort results:

Alphabetically

By highest or lowest population

By most recent orders

By product price

Sorting helped create clear, meaningful reports for stakeholders.

### Aggregations with GROUP BY
Using GROUP BY, I produced summary insights such as:

Number of products per category

Average population per country

Frequency of city names

Total quantities ordered

These aggregations supported trend analysis and operational reporting.

### Combining Data with JOINs
I used multiple JOIN types to connect related tables:

INNER JOIN – e.g., linking products to suppliers

LEFT JOIN – e.g., listing all courses even without enrolments

RIGHT JOIN – e.g., retrieving all enrolment records

FULL JOIN (conceptual) – understanding complete dataset coverage

CROSS JOIN – generating all possible combinations

JOINs enabled multi‑table insights such as:

Complete order overviews

Supply chain reports

Learner‑course relationships

Capital city comparisons
